# Fake Engagement Behavioural Analytics System

Detect artificial social media engagement using behavioural pattern analysis, anomaly detection, and explainable AI.

---

## Overview

Social media platforms increasingly suffer from artificial engagement generated by bots or coordinated activity. Such behaviour distorts authentic interaction patterns and misleads analytics systems.

This project presents a behavioural analytics system that detects artificial engagement by analysing temporal interaction patterns, identifying anomalies, and computing authenticity scores with explainable behavioural triggers.

The system provides:

- Authenticity Score (0–100)
- Bot Probability
- Behavioural Anomaly Detection
- Engagement Timeline Visualization
- Synthetic Behaviour Simulation
- Interactive Web Dashboard



## Problem Statement

Differentiate organic versus artificial engagement in time-series interaction data by detecting behavioural anomalies such as:

- timing regularity  
- engagement bursts  
- coordinated patterns  
- abnormal spikes

## Dataset

**Dataset Type:** Synthetic Behavioural Engagement Data

### Why Synthetic

No publicly available dataset contains labelled artificial engagement behaviour with temporal interaction patterns and coordinated anomalies. Synthetic simulation enables controlled modelling of behavioural fraud scenarios.

### Generation Method

Engagement time-series are generated using behavioural rules:

**Normal behaviour**
- stochastic interaction rates  
- circadian activity cycles  
- natural variability  

**Artificial behaviour injection**
- sudden engagement bursts  
- periodic coordinated activity  
- spike after inactivity  
- abnormal timing regularity  

### Assumptions

- Bots produce temporally clustered activity  
- Artificial engagement shows lower entropy  
- Coordinated campaigns exhibit periodic patterns  
- Authentic engagement has natural variance  

### Dataset Size

Configurable during simulation:

- Time duration: user-defined  
- Accounts: user-defined  
- Typical demo: 5k–50k timepoints  

### Feature Description

Extracted behavioural features:

- engagement_rate  
- burstiness_index  
- periodicity_score  
- inter_event_variance  
- spike_magnitude  
- post_idle_jump  
- rolling_activity_change  

These features capture behavioural anomalies indicative of artificial engagement.

---

## Feature Engineering Logic

Behavioural indicators derived from time-series analysis:

- Burst detection via local intensity gradients  
- Periodicity via autocorrelation analysis  
- Variance spikes via rolling deviation  
- Post-idle jumps via inactivity segmentation  
- Temporal regularity via entropy estimation  

These features transform raw engagement counts into behavioural signals suitable for anomaly detection.

---

## Model

**Type:** Unsupervised Anomaly Detection

**Approach**

- behavioural feature vector construction  
- anomaly scoring via trained classifier  
- normalization to authenticity score  

**Outputs**

- anomaly_score  
- authenticity_score  
- bot_probability  

---

## Authenticity Scoring

Anomaly score is mapped to behavioural authenticity:

- **70–100 → Organic Engagement**  
- **30–70 → Suspicious Behaviour**  
- **0–30 → Artificial Engagement**  

Bot probability is derived from normalized anomaly magnitude.

---

## Behavioural Explanation

Model outputs are translated into human-interpretable triggers:

- Sudden engagement burst  
- Coordinated periodic activity  
- Abnormal spike after inactivity  
- Timing regularity anomaly  

These explanations provide actionable behavioural insight.
